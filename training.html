Cas pratique : serveur Ubuntu 24.10 et surveillance SSH

Projet stage : configuration d’un serveur Ubuntu 24.10 pour gérer des caméras via SSH avec Shinobi et ajout d’une caméra Axis via Axis IP Utility.

Inspiré du document de stage : documentationsrvCaméra02final.pdf.

UbuntuShinobiAxis

Étapes clés illustrées

Préparation du serveur, installation de Shinobi, configuration réseau, ajout de caméra Axis, tests, et production de documentation technique.

DocumentationTestsSécurité

Synthèse PractiKa : étapes du projet

  1. Installer Ubuntu 24.10 sur le serveur et sécuriser l'accès SSH.
  2. Configurer Shinobi pour capturer et gérer les flux des caméras.
  3. Ajouter la caméra Axis avec Axis IP Utility et vérifier la connectivité réseau.
  4. Tester le flux vidéo en local et à distance, puis ajuster les permissions et les ports.
  5. Documenter le processus dans le rapport de stage et tirer les leçons pour optimiser l’installation.

Pourquoi PractiKa fonctionne

PractiKa combine la théorie et l’action : vous apprenez en faisant, vous corrigez en temps réel et vous formalisez vos résultats. Ce cycle renforce la compréhension et prépare à reproduire le même processus sur de nouveaux projets.

Conseils PractiKa :

  • Notez chaque commande et chaque paramètre important.
  • Capturez les erreurs rencontrées et les solutions appliquées.
  • Reliez les résultats aux objectifs du projet de stage.
  • Réutilisez la même structure pour documenter d’autres configurations.
